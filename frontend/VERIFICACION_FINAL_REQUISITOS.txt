🔄 FLUJO DE DATOS

1. Usuario abre app
   → App.js envuelto en <ThemeProvider>
   → Lee localStorage para tema
   → Aplica clase "dark" si necesario

2. Usuario navega a proyecto
   → ProjectsPage carga lista de proyectos
   → Click en proyecto → /proyecto/:id
   → ProjectDetailPage carga proyecto con ID

3. Usuario expande alternativa
   → Click en AlternativaCard
   → Se expande para mostrar DocumentosList
   → Muestra botón "+ Subir"

4. Usuario sube documento
   → Click "+ Subir"
   → Se abre DocumentoUploadModal
   → Drag & drop o click
   → Post a API
   → Aparece en lista

5. Usuario cambia tema
   → Click en ThemeToggle
   → toggleTheme() en context
   → Se guarda en localStorage
   → Se aplica inmediatamente
   → Persiste en próxima sesión

📝 NOTAS IMPORTANTES

1. Backend URL: http://127.0.0.1:8000/api
   - Debe estar corriendo en Django
   - CORS debe estar habilitado
   - Endpoints esperados (ya en api.js)

2. localStorage: Debe estar habilitado
   - Tema se persiste aquí
   - Revisa DevTools → Application → localStorage

3. Tailwind Dark Mode: Usa clase "dark"
   - Se aplica a <html class="dark">
   - Configuración: darkMode: 'class' en tailwind.config.js

4. Context API: ThemeProvider en root
   - Todos los componentes pueden usar useTheme()
   - Evita prop drilling

5. Componentes Reutilizables:
   - AlternativaCard es mapeado en lista
   - Modals se pueden extender
   - ThemeToggle en cualquier header
